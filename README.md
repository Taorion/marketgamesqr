# UNIVERSAL QR / Plataforma de activacion, leads y redencion

## Resumen ejecutivo

UNIVERSAL QR es una plataforma B2B para crear, validar, redimir y medir QR asociados a campañas, leads, beneficios y ventas atribuidas.

La plataforma convierte una activacion promocional en un sistema medible de:

- captura de leads;
- emision de QR;
- validacion en punto de venta;
- redencion controlada;
- auditoria;
- analitica comercial.

El producto no se vende como un "juego". Se vende como infraestructura comercial para transformar trafico en datos, redenciones y ventas medibles.

## Como entender UNIVERSAL QR

Si un modelo como ChatGPT necesita interpretar este proyecto, debe asumir lo siguiente:

1. `MotoPescuezo` es una experiencia de entrada, no el negocio principal.
2. El negocio real es UNIVERSAL QR, la infraestructura de activacion comercial con trazabilidad.
3. El valor principal esta en conectar:
   - experiencia digital,
   - captura de datos,
   - emision de QR unico,
   - validacion en tienda o punto de entrega,
   - redencion segura,
   - analitica de conversion.
4. El sistema esta pensado para operar con multiples negocios y multiples campañas.
5. La utilidad para una empresa es que convierte promociones dispersas en un funnel medible.

Formula corta del producto:

```text
Campana digital + lead capturado + QR unico + validacion + redencion + analitica = promocion rentable y auditable
```

## Que hace el producto

La plataforma permite que una marca, tienda, restaurante, retail, distribuidor o agencia ejecute campañas donde una persona:

1. participa en un juego, formulario, landing, encuesta o activacion;
2. deja sus datos;
3. recibe un QR unico;
4. visita un punto fisico o reclama un beneficio;
5. un vendedor o validador revisa el QR;
6. el sistema registra validacion, redencion y, si aplica, la venta atribuida.

Esto crea un flujo completo entre marketing, operacion comercial y medicion.

## Problema que resuelve

La mayoria de promociones tradicionales tienen varios problemas:

- no se sabe quien participo realmente;
- no hay trazabilidad entre campaña y venta;
- el mismo beneficio puede redimirse varias veces si no hay control;
- el negocio captura pocos datos utiles;
- no hay forma simple de validar identidad o vigencia;
- se hacen activaciones bonitas, pero no auditables.

UNIVERSAL QR resuelve eso con:

- token unico;
- estados de QR;
- validacion centralizada;
- redencion controlada;
- logs de auditoria;
- reportes por campaña y por negocio.

## Componentes del repositorio

### 1. Experiencia de entrada

`MotoPescuezo` es el ejemplo funcional de microexperiencia promocional conectada al backend. Sirve como puerta de entrada para generar leads y emitir QR dentro de UNIVERSAL QR.

Archivos principales:

```text
index.html
css/styles.css
js/game.js
```

### 2. Backend

API en Node.js y Express para:

- crear QR de campanas publicas;
- guardar jugadores y leads;
- administrar negocios y campanas;
- validar y redimir QR;
- exponer reportes;
- soportar portal empresarial y admin.

Entrada principal:

```text
backend/src/server.js
backend/src/app.js
```

### 3. Validador universal

Interfaz para vendedores o personal operativo.

Ruta publica local:

```text
/validador
/qr-validador
```

`/qr-validador` expone el mismo validador standalone como link dedicado para personal operativo que no debe entrar al portal de `/empresa`. La ruta `/validador` se conserva para compatibilidad con QR existentes.

Funciones:

- login de empleado;
- lectura de QR;
- ingreso manual de token;
- consulta de estado;
- visualizacion de nombre, cedula, telefono y beneficio;
- redencion unica.

### 4. Portal empresarial

Interfaz para clientes de negocio.

Ruta publica local:

```text
/empresa
```

Funciones reflejadas en API:

- ver campanas;
- consultar reportes;
- ver leads;
- ver redenciones;
- descargar QR activos;
- registrar snapshots de ventas;
- operar el validador desde el portal;
- registrar ventas atribuidas despues de una redencion;
- generar QR postventa;
- crear paquetes de QR estrategicos para etiquetas, empaques, volantes, eventos o punto de venta;
- revisar historial de QR estrategicos;
- crear afiliados del negocio;
- subir logo del negocio;
- generar y descargar carnet PNG de afiliado con logo del negocio, puntos y QR permanente;
- sumar puntos por compra al afiliado.

El modulo de afiliados funciona como fidelizacion: el QR del carnet identifica al cliente y permite acumular puntos, no redime un premio por si solo.

### 5. Panel administrativo

Interfaz / API para la operacion central.

Ruta publica local:

```text
/admin
```

Funciones reflejadas en API:

- crear negocios;
- crear campanas;
- editar campanas;
- marcar campanas listas;
- ver resumenes por negocio;
- consultar reportes globales.

### 6. Campana de preferencias de producto

Este repo tambien expone una experiencia para capturar interes comercial sobre productos.

Ruta publica local:

```text
/campana-productos
```

## Arquitectura funcional

```text
Juego / landing / formulario / activacion social
        |
        | POST /api/public/...
        v
Backend Express
        |
        | Guarda lead, campana, respuestas y QR
        v
PostgreSQL / Supabase
        |
        | Genera token y estado del QR
        v
QR entregado al usuario
        |
        | GET /api/qr/validate/:token
        | POST /api/qr/redeem/:token
        v
Validador en tienda
        |
        v
Portal y reportes para negocio / admin
        |
        | QR postventa / paquetes estrategicos / afiliados
        v
Fidelizacion, recompra y ventas atribuidas
```

## Tesis del negocio

La razon por la que este producto puede ser valioso y rentable es simple:

- los negocios siempre necesitan captar clientes;
- las promociones funcionan mejor cuando hay incentivo claro;
- los incentivos fallan cuando no hay control;
- la data vale mas cuando conecta marketing con redencion real;
- si una empresa puede medir eso, puede volver a comprar.

En otras palabras: este proyecto no vende entretenimiento. Vende conversion, control y evidencia.
